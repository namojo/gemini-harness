# 코드 보안 리뷰 리포트 (진행 중)

## 1. 개요
- **리뷰 목적**: 프로젝트 내 Python(`.py`) 파일의 잠재적 보안 취약점 식별 및 안전한 코딩 가이드라인 제시
- **상태**: 파일 탐색 및 수집 중 (대상 파일 확보 대기)

## 2. 보안 리뷰 관점 (3대 요소)
1. **인젝션 위험 (Injection)**: SQL Injection, OS Command Injection 등 신뢰할 수 없는 입력값에 대한 검증 및 안전한 처리 여부 확인
2. **인증 및 인가 (Auth & Access Control)**: 부적절한 권한 검증, 세션 관리 취약점, IDOR(안전하지 않은 직접 객체 참조) 등 접근 통제 결함 확인
3. **민감 데이터 노출 (Data Exposure)**: 하드코딩된 비밀번호/API 키, 중요 정보의 평문 저장 및 전송, 부적절한 에러 핸들링 및 로깅 등 확인

## 3. 상세 리뷰 결과
